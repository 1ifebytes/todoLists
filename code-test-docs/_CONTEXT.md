### ✅ Latest Refactoring: Authentication Decoupling (Complete)

A major architectural refactoring has been completed to decouple authentication concerns from Controller and Service layers:

**Before:**
- Controllers needed `@AuthenticationPrincipal UserPrincipal principal` parameter
- Services required `callerId` as first parameter
- Tests required complex `UserPrincipal` mocks

**After:**
- Controllers: No principal parameter, service calls are cleaner
- Services: Use `UserContext.getCurrentUserId()` internally
- Tests: Simple `UserContext.setCurrentUserId("u1")` setup

**Implementation:**
1. `UserContext.java` - ThreadLocal utility class
2. `JwtAuthenticationFilter.java` - Sets/clears ThreadLocal in finally block
3. All Service interfaces - Removed `callerId` parameters
4. All Service implementations - Use `UserContext.getCurrentUserId()`
5. All Controllers - No `@AuthenticationPrincipal` parameters
6. All tests - Updated with UserContext pattern

## Key Technical Decisions

### 1. Authentication Decoupling
**Decision:** Use ThreadLocal + Filter instead of passing principal through layers
**Rationale:**
- Cleaner method signatures
- Better separation of concerns
- Simpler tests
- Memory leak protection via finally block

### 2. Soft Delete Pattern
**Decision:** Use `deleted_at` DATETIME instead of `is_deleted` BOOLEAN
**Rationale:**
- Auditing capabilities (when was it deleted?)
- Recovery possibility (undo delete)
- Automated cleanup policies (delete after 30 days)
- Natural prevention of duplicate deletes

### 3. UUID vs Snowflake IDs
**Decision:** Use UUID generated by application layer
**Rationale:**
- Distributed system friendliness
- No coordination required
- Sufficient for expected scale
- Security: IDs are not predictable

### 4. Field-level LWW (Last Write Wins)
**Decision:** Conflict resolution at field level, not row level
**Rationale:**
- Prevents false conflicts when users edit different fields
- More intuitive collaborative experience
- Simpler implementation

### 5. ThreadLocal for User Context
**Decision:** Store current user in ThreadLocal for request duration
**Rationale:**
- Avoids passing callerId through every layer
- Thread-safe for synchronous applications
- Automatic cleanup via Filter's finally block

## Critical Gotchas

1. **Docker MySQL Port**: MySQL runs on port **3307** (mapped from 3306 inside container)
2. **Testcontainers Skipping**: 9 tests skipped due to Docker Desktop 29 API version compatibility (known issue, not a code problem)
3. **UserContext Cleanup**: Always call `UserContext.clear()` in tests (use try-finally)
4. **ActivityFeedService.record()**: Has explicit `actorId` parameter (for recording other users' actions)
5. **Lombok + MapStruct Order**: MapStruct must be AFTER Lombok in annotationProcessorPaths
6. **Mockito Strict Mode**: Use `any()` for void methods with POJOs without `equals()`
7. **@SQLRestriction**: Use Hibernate 6 syntax (not `@Where` from Hibernate 5)
8. **Logger Name**: Use `com.sleekflow` package name (not old `com.lifebytes`)
9. **Collection Builder Fields**: Use `@Builder.Default` or initialize inline (not field init with @Builder)
